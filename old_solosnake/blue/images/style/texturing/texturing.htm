This is the current default view on to the board, always showing the full extent of the board. This board is only 24 x 24 tiles, but boards can be up to a max of 100 x 100.
How many bots a board can be playable with is currently limited mostly by the graphics card. The game is about a LOT of bots, not a handful.

This is a view showing a cluster of bots a user might be interested in.
They can almost see what components each bot is built from.
They will be able to tell what they are doing when SFX and GUI feedback are added.
Which emissives are active, as well as how bright and animated they are will also give a uer visual feedback about a bot's state.

A closer look at the bot cluster.
Now the user can make out each component of the bots and assess them better.

At this zoom level the user can examine a bot in great detail.
However this zoom level is for curiousity or admiring assets only - it has no gameplay use.
Zooming closer than this may be disabled by the game.

This is a mimap analysis render of the bot.
The bot has 23 components, and each component is using a set of three 512 x 512 RGBA textures.
Only the surfaces of the bot which are rendererd in pure red are actually using the top level 512 x 512 mipmap.
The orange surfaces are rendered from the down-sampled 256 x 256 mipmap
Etc down through yellow (128 x 128), green (32 x 32) and blue (16 x 16).

This is a mimap analysis render of the scene.
A lot of the scene is rendered with the Blue (16 x 16), Indigo (8 x 8) and Violet (4 x 4).
Other colors appear becuase the final mipmap (1 x 1) is white and reflects the local light color.
